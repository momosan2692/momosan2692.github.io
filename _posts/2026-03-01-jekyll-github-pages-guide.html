Step-by-step guide

GitHub Pages
+ Jekyll on macOS

Free blog hosting with Markdown support — no server required. Everything runs on GitHub.

✦ 100% Free ✦ Markdown Native ✦ Custom Domain macOS
Prerequisites
1
Create a GitHub Account

Go to github.com and sign up for a free account if you don't have one yet. Your blog URL will be based on your username: yourusername.github.io

2
Install Homebrew (macOS package manager)

Open Terminal (press ⌘+Space, type "Terminal") and run:

bash 
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
Tip: If you already have Homebrew, skip this step. Check by running brew --version
3
Install Ruby and Jekyll

macOS has Ruby built-in, but it's better to install a fresh one via Homebrew:

bash 
# Install Ruby
brew install ruby

# Add Ruby to your PATH (add this to ~/.zshrc or ~/.bash_profile)
echo 'export PATH="/opt/homebrew/opt/ruby/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

# Install Jekyll and Bundler
gem install jekyll bundler
Create your site
4
Create a New Jekyll Site

In Terminal, navigate to where you want your blog folder and run:

bash 
# Create new Jekyll site
jekyll new my-blog

# Enter the folder
cd my-blog

# Preview locally in your browser
bundle exec jekyll serve
Tip: Open http://localhost:4000 in your browser to see your site live locally!
5
Write Your First Blog Post

Create a new file in the _posts/ folder. The filename format must be: YYYY-MM-DD-title.md

_posts/2026-03-01-my-first-post.md
---
layout: post
title:  "My First Blog Post"
date:   2026-03-01 12:00:00 +0800
categories: blog
---

# Hello World!

This is my first post written in **Markdown**.

- Item one
- Item two

> A great quote goes here.

Happy blogging! 🎉
Publish to GitHub
6
Create a GitHub Repository

On GitHub, click New Repository and name it exactly:
yourusername.github.io

Then push your local Jekyll site to GitHub:

bash 
# Initialize git in your project folder
git init
git add .
git commit -m "Initial blog setup"

# Connect to GitHub (replace YOUR_USERNAME)
git remote add origin https://github.com/YOUR_USERNAME/YOUR_USERNAME.github.io.git

# Push!
git branch -M main
git push -u origin main
7
Enable GitHub Pages

On your GitHub repository page, go to Settings → Pages. Under "Source", select Deploy from branch → main → / (root) and click Save.

⏳ Wait ~2 minutes — then visit https://yourusername.github.io and your blog is live! 🎉
🎉 Your blog is live!

Here's what you got for free:

  • Free hosting at yourusername.github.io
  • Write posts in Markdown (.md files)
  • Auto-rebuild when you push new posts
  • Free HTTPS / SSL certificate
  • Can connect your own custom domain
