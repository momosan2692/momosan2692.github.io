# U-ViT vs DiT vs CogArch——三條架構路線的數學底層，與物理引擎的設計哲學

> 世界模型系列 #2 · April 2026  
> Covers: U-ViT 數學實作、DiT adaLN-Zero、CogVideoX 3D-VAE、推理效率優化、成本影響

---

## 1. 三條路線的起源與哲學分野

理解當前視訊生成架構的競爭格局，必須先理解三條技術路線在設計哲學上的根本分歧。

**U-ViT**（生數科技 / Vidu 的基石）的哲學是：**「視覺即序列，時空即語言。」** 它徹底拋棄了傳統視訊生成的 CNN 骨架，把影像的每一個切片（Patch）、每一個時間步、每一個文字條件，全部轉化為同一維度的 Token，讓 Transformer 的自注意力機制在這個統一的序列上自由地建立任意長程關聯。

**DiT**（OpenAI Sora 的核心，Peebles & Xie 2022）的哲學是：**「暴力 Scaling，讓模型自己學會一切。」** 它採用線性堆疊的 Transformer block，不引入任何手工設計的結構偏置（Inductive Bias），完全依賴參數規模的擴展（Scaling Law）來讓模型湧現出物理直覺。

**CogVideoX**（智譜 AI）的哲學是：**「用語言的智慧引導視覺的生成。」** 它繼承了 GLM 系列大語言模型對複雜語義的深度理解，透過 3D-VAE 將時空信息壓縮到一個高效的潛在空間，再由語言模型的邏輯推理能力驅動像素的生成。

這三種哲學，導致了截然不同的技術選擇，也各自帶來了截然不同的成本結構。

## 2. U-ViT 的數學底層：為什麼「跳躍連接」是不崩壞的保證

生數科技的技術基礎論文是 2023 年 CVPR 收錄的《All are Worth Words: A ViT Backbone for Diffusion Models》。這篇論文的核心貢獻，是證明了純 Transformer 架構在擴散模型中，不只可行，而且在物理動態的連貫性上系統性地優於傳統 U-Net。

### 2.1 時空 Token 化

在 U-ViT 中，輸入的潛在表示 $z_t$（視訊在潛在空間中的編碼）首先被切分為固定大小的 Patch，然後展平並線性投影為 Token 向量：

$$X_{\text{patch}} = \text{Linear}(\text{Flatten}(z_t))$$

關鍵在於，時間步信息 $T$ 和文字條件 $C$ 並不是以「旁路調製」的方式注入，而是被直接**當作序列中的普通 Token** 拼接進去：

$$\text{Sequence} = [T_{\text{token}};\ C_{\text{token}};\ P_1,\ P_2,\ \dots,\ P_n]$$

這個設計讓模型在計算自注意力時，每一個畫面切片都能**直接與時間信息和文字指令進行注意力計算**，而非間接接受調製。效果是：模型對「第幾秒的畫面應該呈現什麼」的感知，比旁路注入的方式精確得多。

### 2.2 U 型對稱結構與跳躍連接

U-ViT 的命名來自其採用了類似 U-Net 的對稱結構，但骨幹全部替換為 Transformer block。這是它與標準 ViT 最關鍵的結構差異。

整個計算過程分為三段：

- **Encoder 段（淺層）**：提取高頻局部特徵——物體的邊緣、紋理、位置。
- **Bridge 段（深層）**：處理全局語義——這個物體是什麼、它正在做什麼、與其他物體的空間關係。
- **Decoder 段（對稱淺層）**：將全局語義映射回具體的像素空間，重建最終影像。

跳躍連接（Skip-connection）將 Encoder 第 $L$ 層的輸出直接與 Decoder 對稱第 $M$ 層的輸入進行拼接（Concatenate），再通過線性投影：

$$h_{\text{decoder}} = \text{Linear}(\text{Concat}(h_{\text{encoder}},\ h_{\text{bridge}}))$$

**這個設計解決了長視訊一致性問題的根本原因**：淺層 Encoder 保存了物體的精確位置、形狀、顏色等低層特徵，在經過深層的語義計算後，Decoder 透過跳躍連接能直接取用這些低層特徵，而不需要從語義層重新「回憶」物體的外觀細節。在長視訊生成中，這防止了模型在深層計算後「忘記」物體的初始狀態。

### 2.3 全域自注意力：長程依賴的數學保證

U-ViT 採用標準的 Scaled Dot-Product Self-Attention：

$$\text{Attention}(Q, K, V) = \text{Softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

與卷積的局部感受野（受卷積核大小限制，通常只能看到鄰近幾個像素）不同，自注意力的感受野是**全局的**：序列中的任意一個 Token，都能直接與序列中的任意其他 Token 計算相關性。

物理意義是：模型在生成第 60 秒的畫面時，可以直接查詢第 1 秒時物體的 Query 向量（$Q$）與 Key 向量（$K$）之間的匹配關係，數學上保證了跨時間的物體恆常性。

### 2.4 訓練目標

U-ViT 遵循標準擴散模型的訓練目標——預測在時間步 $t$、文字條件 $c$ 下加入的噪聲 $\epsilon$：

$$\min_\theta\ \mathbb{E}_{z,\, \epsilon,\, t,\, c}\left[\left\|\epsilon - \epsilon_\theta(z_t,\ t,\ c)\right\|^2\right]$$

當模型參數 $\theta$ 擴展到十億（Billion）級別以上時，預測噪聲的能力會出現「湧現（Emergence）」——模型開始能準確預測受重力支配的彈跳序列、受流體力學支配的煙霧擴散，因為這些物理規律在統計意義上是視訊數據中最強的結構信號。

## 3. DiT（Sora 路線）：線性堆疊與 adaLN-Zero

DiT 的結構相對 U-ViT 更為簡潔：沒有跳躍連接，純粹的線性 Transformer block 堆疊。它的精妙之處在於**條件注入的方式**以及**初始化策略**。

### 3.1 adaLN-Zero：超大規模訓練穩定性的秘密

DiT 採用**自適應層歸一化（Adaptive Layer Normalization，adaLN）**來注入條件信息（時間步 $t$ 和文字嵌入 $c$）。與 U-ViT 將條件作為 Token 加入序列不同，DiT 將條件變換為兩個調製參數：縮放因子 $\gamma$ 和偏移量 $\beta$，直接作用於每一個 Transformer block 的歸一化層：

$$\text{AdaLN}(h,\ t,\ c) = \gamma(t, c) \cdot \text{LayerNorm}(h) + \beta(t, c)$$

adaLN-**Zero** 的關鍵創新在於初始化策略：所有 $\gamma$ 的初始值設為 0。這讓每一個 DiT block 在訓練開始時等效於一個**恆等映射（Identity function）**——信號完整地穿透 block 而不做任何修改。

這個設計在超大規模訓練中至關重要。當 DiT 的深度達到 28 層甚至更多時，如果每一層初始時都對信號施加隨機的縮放，梯度在反向傳播時會迅速爆炸或消失。adaLN-Zero 的恆等初始化確保了訓練初期的梯度流動完全穩定，使 OpenAI 能夠將 Sora 推進到超深的架構而不遭遇訓練崩潰。

U-ViT 因為引入了跳躍連接，梯度可以透過殘差路徑直接流動，因此不需要這個技巧。但這也意味著在**超深架構的訓練穩定性**上，純 adaLN-Zero 的 DiT 理論上具有系統性優勢，這是 Sora 在超大參數規模上能成立的工程基礎之一。

### 3.2 DiT 的設計代價

DiT 的線性堆疊沒有跳躍連接，意味著淺層的局部特徵信息需要完全依靠深層計算來「記住」——這在短視訊（< 10 秒）中問題不大，但在長視訊（> 60 秒）中，物體的細節特徵容易在極深的計算路徑中逐漸「稀釋」。這是 DiT 路線在長視訊物理一致性上需要靠大量參數彌補的結構性挑戰。

## 4. CogVideoX：3D-VAE 與語言引導的哲學

智譜 AI 的 CogVideoX 走了一條與前兩者明顯不同的路線，其核心在於**先壓縮，再理解，再生成**的三段式流程。

### 4.1 3D-VAE 的時空壓縮

3D 變分自編碼器（3D-VAE）將視訊在時間和空間兩個維度同時進行壓縮，把一個「$T$ 幀 × $H$ × $W$ 像素」的視訊編碼進一個遠更小的潛在空間。

典型的壓縮比是**時空各壓縮 4 倍**，即 $4 \times 4 \times 4 = 64$ 倍的體積縮減。這意味著一段 64 幀的 256×256 視訊，在潛在空間中只需要處理 $16 \times 64 \times 64 = 65536$ 個值，而非 $64 \times 256 \times 256 \approx 4M$ 個原始像素值。對計算量的影響是直接的：在潛在空間中運行 Transformer 的計算量約為原始像素空間的 **$1/64^{2/3} \approx 1/16$**。

### 4.2 語義繼承：從 GLM 到 CogVideoX

CogVideoX 的技術路線強調，視覺生成不應該「從零學習物理」，而應該「繼承語言模型對世界的理解」。其訓練策略包含了從 GLM 系列預訓練模型遷移語義表徵的步驟，使得視訊生成模型在面對複雜指令時，能調用語言模型的邏輯推理能力來解析指令的因果結構。

這在處理「一位宇航員在火星表面駕駛探測車，揚起橙色沙塵」這類指令時效果尤為突出——模型能正確解析宇航員、火星重力（較低）、沙塵（較細、擴散速度不同）之間的多重約束關係，生成物理上自洽的畫面。

## 5. 三架構全面對比

| 維度 | U-ViT（生數 / Vidu） | DiT（Sora 路線） | CogVideoX（智譜） |
|---|---|---|---|
| **骨幹結構** | U 型對稱 + 跳躍連接 | 線性堆疊 | 3D-VAE + Transformer |
| **條件注入** | Token 拼接（更靈活） | adaLN-Zero（更穩定） | 語言模型語義遷移 |
| **感受野** | 全域（全序列注意力） | 全域（全序列注意力） | 壓縮潛在空間全域 |
| **長視訊一致性** | ⭐⭐⭐⭐⭐（跳躍連接直接保護低層特徵） | ⭐⭐⭐（依賴深度彌補） | ⭐⭐⭐⭐（壓縮後一致性好） |
| **指令理解力** | ⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐⭐（語言模型加持） |
| **超大規模訓練穩定性** | ⭐⭐⭐⭐（跳躍連接保梯度） | ⭐⭐⭐⭐⭐（adaLN-Zero） | ⭐⭐⭐⭐ |
| **計算效率** | 中（全域注意力昂貴） | 低（無壓縮，全域注意力） | 高（VAE 壓縮大幅降低序列長度） |
| **最適合的任務** | 物理動態模擬、長視訊 | 超高清短片、極致細節 | 複雜敘事指令、長片生成 |

**核心結論**：三條路線的優劣不是絕對的，而是在不同任務需求下的**設計哲學取捨**。如果目標是「讓 AI 理解物理世界的運行規律」，U-ViT 的架構是目前最接近的路徑；如果目標是「讓 AI 精確執行複雜的導演指令」，CogVideoX 的語言引導架構更有優勢；如果目標是「靠 Scaling 把極致細節的品質推到天花板」，DiT 的路線有理論上最強的擴展性。

## 6. 推理效率優化：成本壓縮的工程側

架構選擇不只影響生成品質，更直接決定**運行成本**。視訊模型的核心計算瓶頸是自注意力的 $O(N^2)$ 複雜度——當序列長度 $N$ 翻倍時（更長的視訊、更高的解析度），計算量增加四倍，顯存佔用也增加四倍。以下是業界主流的四類優化技術：

### Flash Attention v2 / v3：IO 優化，非計算量優化

Flash Attention 不改變自注意力的數學結果，而是透過重新安排 GPU 的 SRAM 與 HBM 之間的讀寫順序，大幅減少顯存 IO 次數。具體地，它將 Q、K、V 矩陣分塊（Tiling）加載到 SRAM 中完成計算，避免了每一步都需要從 HBM 讀取完整矩陣的開銷。

效果是：速度提升 **2–4×**，顯存佔用降低 **5–20×**，且這個優化對所有使用標準自注意力的架構（U-ViT、DiT、CogVideoX）都適用。這是 2024–2026 年間大陸所有頭部視訊模型的**必選優化項**。

### 3D-VAE 時空壓縮：最大化 Token 縮減

如前所述，3D-VAE 在時空兩個維度上同時壓縮，典型壓縮比 64 倍。計算量的縮減不是線性的：對於自注意力的 $O(N^2)$，Token 數量縮減 8 倍意味著計算量縮減 **64 倍**。這是 CogVideoX 能夠以相對較少的算力訓練較長視訊的核心原因。

### Flow Matching vs DDPM：訓練步數的壓縮

傳統擴散模型（DDPM）需要 50–1000 步去噪才能生成高質量樣本。Flow Matching 將加噪到去噪的路徑從曲折的布朗運動改為**直線流（Straight Flow）**，訓練更穩定，且推理只需 **10–20 步**即可達到 DDPM 50 步的質量。生數科技的 Vidu 採用了 Flow Matching，這直接將其推理速度提升了約 3–5 倍，也是其 API 能夠維持較低定價的技術基礎之一。

蒸餾技術（如 Consistency Distillation、LCM）可以進一步壓縮到 **1–4 步**，但會有一定的質量損失，通常用於「快速預覽」模式而非最終輸出。

### Ring Attention：突破單卡顯存限制的長視訊解決方案

對於分鐘級的超長視訊，即使使用了 Flash Attention 和 3D-VAE 壓縮，單卡 80GB 的顯存仍然不足以容納完整的序列。Ring Attention 將超長序列**分片分配到多張 GPU 上**，每張 GPU 只計算自己負責的片段，再透過環狀（Ring）的 AllGather 通訊協議讓每張卡都獲得其他卡的 KV Cache，最終等效於在一個統一的序列上計算全域注意力。

這是 Sora 生成分鐘級長視訊的推定技術手段，也是大陸廠商在萬卡叢集上突破長視訊一致性瓶頸的關鍵工程選擇。

## 7. 架構選擇如何影響 API 成本

優化技術不只是技術性能問題，它直接決定了**商業定價能力**。以下是各優化技術對 API 成本的量化影響：

| 優化技術 | 訓練吞吐量提升 | 推理成本降低 | 備註 |
|---|---|---|---|
| Flash Attention v2/v3 | +2–4× 速度 | −50–75% | 適用所有架構 |
| 3D-VAE（典型 64×壓縮） | Token 數 ÷8，計算量 ÷64 | −90%+ | 僅視訊模型 |
| Flow Matching（vs DDPM） | 步數 ÷5，等效速度 +5× | −70–80% | 訓練+推理雙重受益 |
| MoE 稀疏激活（激活 1/8） | 等效算力 +4–6× | −60–75%（視稀疏度） | 適用大模型推理 |
| 蒸餾（4 步 vs 50 步） | 推理步數 ÷12 | −80%+ | 有質量損失 |

疊加使用時，整體 MFU（Model FLOPS Utilization）可從未優化的 30–40% 提升到 55–65%。換句話說，相同的硬體投入，在算法優化後等效算力幾乎翻倍——這解釋了智譜 API 在提價 83% 的同時，呼叫量仍增長 400% 的現象：供給側的成本下降速度快於定價下降速度，毛利率因此從 2024 年的低個位數擴張到 2025 年的 18.9%，再到 2026 年繼續走高。

## 8. 下一篇預告：當架構遇到物理限制

再精妙的架構，也需要算力來驅動。當 NVIDIA 的出口管制將最強算力擋在門外，大陸廠商如何在「H20 算力僅為 H100 的 15%」的硬體限制下，仍然推進世界模型的訓練？答案不只是「堆更多卡」，而是一場從網路拓撲到通訊協議的**全棧工程優化戰**。

**本系列第三篇**將深入萬卡叢集的網路工程學——從 RoCE v2 的死穴到自研擁塞控制算法，從三級並行的拓撲映射到帶內遙測的閉環故障隔離，再到最終的訓練與推論成本壓縮路線圖。
